# 更新日志

## v0.9.2
- **再战拦截神之锅**：经测试确认原生 `PermissionType.ADMIN` 会漏放给大模型。重新在模块底层手动下发 `_is_admin` 权限盾。
- **兼容艾特截断**：新增对 `[At:]` 解析器的支持。
- **数据库级灵魂容器（永久人设 RAG 机制）**：重磅功能上线！引入了真正的底层记忆体系。
  - 新增 `/sadstory_capture @张三 [条数]` 指令。系统会在后台疯狂分页追溯最近的群历史消息，精准过滤出只属于该人的数十条发言，交给大模型提纯出一份“灵魂侧写卡”，并**永久固化进 SQLite 数据库**的 `user_personas` 表中。
  - 从此以后，只要此人被设定为小剧场的主角，大模型都会在底层深处强行加载并附体他的专属人设卡。无需每次生成再重新抓取聊天记录，同时消灭了无尽的延迟打底与 Token 开销噩梦！
  - 新增附带指令：`/sadstory_listp` 查看被关在小黑屋的灵魂图鉴；`/sadstory_delp` 将某个灵魂放归自由。

## v0.9.0

## v0.8.6
- **跨服点名系统（管理员专属）**：新增前置纯数字 QQ 解析机制。现在系统管理员可以直接在指令开头输入一串或两串纯数字（如 `/sadstory 12345 67890 聊点什么`），系统将利用底层 `get_stranger_info` API 强行进行跨站穿透，直接拉取该号的网名和真实头像用于合并转发，无需他们加群。普通群员依然只能乖乖艾特同群小伙伴。

## v0.8.5
- **依赖热修复**：紧急修复在重构为 Mixin 架构时，由于 AstrBot 框架动态挂载导致的环境解析错误（`No module named 'configs'`）。将拆分出来的模块间互调全部升级为极其严格的原生防丢失手段（防错的 Package 包形态相对路径导入）。
- **底包健壮性补充**：一并补齐了在指令模块分离时少导的 `logger`, `asyncio`, `json`, `random` 等 Python 基础内库依赖，保证发信端运行不再存在语法层面的硬核异常缺失崩溃。

## v0.8.4
- **架构重构升级**：彻底拆分了解耦了原本长达 1600+ 行的 `main.py` 巨型文件结构（重构为模块化驱动），将文本常量池（Prompt）、业务核心生成算法（Generator）以及数十个子管理指令（Commands）无损迁移到极度扁平优雅的 Python Mixin（多继承拆分体系）树中，后续开发维护极度舒适。
- **互动格式极简回归**：遵从最高还原度的使用体验，彻底抹除大模型强加的“伪造文本边框引用”功能。因为协议物理层级不支持假 Message ID 气泡，伪造的“回复边框文字”极度扎眼，故全面回归到清爽顺眼的「@某人 然后直接回复原文」的最强野生网感姿态。

## v0.8.3
- **强力修复**：重构了底层的贴纸抓取传输代码。现在在跨框架或 Docker 等隔离部署（如 AstrBot 内核与 Napcat 发信端隔离）时，贴纸将直接转换成网络直出的 Base64 二进制流直接塞入！彻底终结了找物理硬盘报错 `ENOENT` 的世纪难题。
- **特性新增**：为大模型开启了解锁“穿插引用”的被动技能点。模型现在能够且强烈倾向于以“「回复某某」”等高仿引用的文字格式回应前言，不再是一味地按顺序平铺直述。
- **配置化特性**：新增“智能削减句末标点”功能（WebUI 可在底层配置）。专门用于斩杀大模型改不掉的非要把话后加个句号的坏毛病，保留逗号，精确保留图文并发场景的同时，大幅度提升口语网感。
